  1. Define HTML

    2. HTML is defined as a markup language.
    A markup language is simply a way of annotating a document in such a way to mkae the annotations distinct from the text being annotated.
    The term comes from the days of print, when editors would write instructions on manuscript pages that might be revision instructions to the author or copy editor.

  2. Define Client-server model of communications.

    3. Client-Server Model describes the communication between the server and clients which request resources from the listening server.

  3. Write down the names of different types of servers.

    1. Data Server

    3. Authentication Server

  4. Define a Web Server

    5. A web server is relates to hardware and/or software working together.
    Web server hardware stores web server software and the website's component files.
    Web server software controls how web users access the hosted files.

  5. Define Data Centers.

    6. Data centers are server farms typically housed in special facilities.
    They are used to prevent the potential for site down times.
    As a consequence, the costs for multiple redundant data centers are quite high, and only larger web comapnies can afford this.
    Most web companies will instead lease space from a third-party center.

  6. Why do we need a Uniform Resource Locator(URL)?

    7. The reason is to allow clients to request particular resources from the server.
    Uniform Resource Locator(URL) is the naming mechanism used.

  7. Give an example of Uniform Resource Locator (URL) and specify all components of an URL.

    8. Example: http://www.website.com/index.php?page=20#about

    1. Protocol: http
    2. Domain: www.website.com
    3. Path: index.php
    4. Query String: page=17
    5. Fragment: about

  8. What is the difference between a static and a dynamic web site?

    9. A Static Web Site consists only of HTML pages that look identical for all users at all times.
    A Dynamic Web Site is when the page content varies between users and is being created at run-time.

  9. Why do we need Domain Name System (DNS)?

    10. Instead of IP addresses, we use the Domain Name System(DNS).

  10. Describe in detail the process of DNS address resolution?

    11. This is the process of obtaining the numeric IP address from the domain name.

    1. Domain name is entered
    2. If IP not in browser's cache, task delegated to OS DNS Resolver
    3. If IP not in DNS cache, resolver requests for IP address to ISP's DNS Server
    4. If primary DNS server doesn't have requested domain in its DNS cache it sends request to root name server
    5. Root name server returns IP of name server for requested TLD
    6. Request IP of name server for domain
    7. .com name server returns IP address of DNS server for domain to Primary DNS server
    8. Primary DNS server request sent for IP address of domain to DNS server
    9. DNS server returns IP address of web server to Primary DNS server
    10. Primary DNS server returns IP address of domain to DNS Resolver
    11. DNS resolver returns IP address of domain to application
    12. Application returns requested page

  14. How can you pass a user's information from the client to the web server?

    15. The HTTP protocol establishes a TCP connection on port 80 (by default).

  15. Describe how a single web page is displayed on a web browser.

    16. The client's browser requests the initial HTML page, then parses the returned HTML to find all resources referenced within such as images, style sheets and scripts.
    The page is fully loaded for the user once all files have been retrieved.
